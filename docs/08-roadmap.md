# 08｜路线图

## Phase 0：概念确认

目标：完成产品定位、MVP 范围、核心循环、知识点地图。

交付物：

- 产品简报。
- MVP PRD。
- 三年级数学知识点地图。
- 任务配置样例。
- 美术风格指南。

## Phase 1：可玩原型

目标：做出 10-20 分钟可玩 Demo。

范围：

- 一个小农场地图。
- 玩家移动和交互。
- 三个 NPC。
- 三类数学任务。
- 一个建筑修复目标。
- 本地 JSON 任务读取。
- 本地存档。

成功标准：

孩子能完成一条主线并看到面包店变化。  
题目不是孤立弹窗，而是由场景触发。  
答错后可以自然重试。  
内容可以通过 JSON 修改。

## Phase 2：MVP 内测版

目标：支持 7 天连续体验。

范围：

- 30-50 个数学任务。
- 日常订单系统。
- 知识点熟练度记录。
- 小镇日记。
- 家长成长报告雏形。
- 内容后台简单版。

成功标准：

孩子 7 日留存有初步数据。  
家长能理解报告。  
教研可以在后台维护任务。  
主线和日常任务形成闭环。

## Phase 3：首个付费成长包

目标：验证商业化。

范围：

- 三年级数学农场季完整包。
- 80-150 个任务。
- 3-5 个建筑修复目标。
- 更完整的学习报告。
- 家长付费购买流程。

## Phase 4：内容扩展

可能方向：

- 小禾镇：唐诗节气季。
- 小禾镇：英语邮局季。
- 小禾镇：五年级工坊季。
- 小禾镇：科学探索季。

扩展原则：

每个内容包都必须先设计“知识点-游戏场景映射”，不能直接导入题库。
